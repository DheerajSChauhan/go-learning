# Contributing to Go Learning

Thanks for your interest in contributing! This repository is intended as a learning resource and reference. Contributions that improve clarity, add examples, or add tests are welcome.

How to contribute
- Fork the repository and create a feature branch: `git checkout -b feat/your-change`.
- Make small, focused commits with clear messages.
- Run `gofmt -w .` and `go vet ./...` before submitting.
- If you add new examples, include a README snippet or comments explaining the intent.
- Add tests where applicable and ensure `go test ./...` passes.

Pull requests
- Open a PR against `main` with a clear title and description.
- Reference any related issue or discussion.
- Keep PRs small and focused to simplify review.

Code style and conventions
- Prefer clear, idiomatic Go. Use `gofmt` for formatting.
- Keep public APIs documented with concise comments.

Reporting issues
- Open an issue with a reproducible example, expected behavior, and actual behavior.

Thank you for improving this learning repository — contributions make it better for everyone.
